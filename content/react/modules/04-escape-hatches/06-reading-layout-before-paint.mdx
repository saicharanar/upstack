Most of the time `useEffect` is exactly right: it runs *after* React updates the DOM **and after
the browser paints**. But that "after paint" is a problem in one specific situation — when you
need to **measure** an element and then adjust the layout based on the measurement. The user
would see the first (wrong) version flash, then the corrected one. `useLayoutEffect` closes that
gap: it runs after the DOM is updated but **before** the browser paints.

## When each one fires

<Mermaid
  chart={`flowchart LR
    R["Render"] --> DOM["React updates the DOM"]
    DOM --> LE["useLayoutEffect runs (measure + adjust)"]
    LE --> Paint["Browser paints — user sees the final result"]
    Paint --> E["useEffect runs (after the user already saw it)"]`}
  caption="useLayoutEffect runs before paint, so measurements and adjustments happen before the user sees anything."
/>

The classic case is a **tooltip** that must know its own height to decide whether to open above
or below the thing it points at. Measure in `useEffect` and it opens in the wrong spot for one
frame, then jumps. Measure in `useLayoutEffect` and the correct position is there from the first
paint — no flicker.

## Measuring a node

```jsx
import { useRef, useState, useLayoutEffect } from 'react';

function Tooltip() {
  const ref = useRef(null);
  const [height, setHeight] = useState(0);

  useLayoutEffect(() => {
    setHeight(ref.current.offsetHeight); // read layout synchronously, before paint
  }, []);

  // ...use height to position the tooltip
}
```

<Callout type="warning" title="Prefer useEffect — this one has a cost">
  `useLayoutEffect` runs **synchronously before paint**, so heavy work here delays what the user
  sees and can make the app feel janky. Reach for it *only* when you measure layout and adjust
  before paint. For everything else — subscriptions, fetching, syncing external systems — use
  `useEffect`.
</Callout>

  code={`import { useRef, useState, useLayoutEffect } from 'react';

export default function Measure() {
  const boxRef = useRef(null);
  const [width, setWidth] = useState(0);

  useLayoutEffect(() => {
    setWidth(boxRef.current.offsetWidth);
  }, []);

  return (

    </div>
  );
}`}

## Recap

- **`useEffect`** runs *after* paint; **`useLayoutEffect`** runs *before* paint.
- Use `useLayoutEffect` when you **measure the DOM and adjust layout** — it prevents the flicker.
- It runs **synchronously** and can hurt performance, so prefer `useEffect` for everything else.
