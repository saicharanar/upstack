React updates the DOM for you, so you rarely touch it directly. But occasionally you need the
**real DOM node** to do something React has no prop for: **focus** an input, **scroll** an
element into view, play a `<video>`, or **measure** an element's size. You reach the node by
attaching a ref to JSX.

## Attaching a ref to a DOM node

```jsx
import { useRef } from 'react';

function Form() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus(); // inputRef.current is the real <input> DOM node
  }

  return (
    <>
      <input ref={inputRef} />
      <button onClick={handleClick}>Focus the input</button>
    </>
  );
}
```

The flow: create a ref with `useRef(null)`, pass it to an element's **`ref`** attribute, and
after React renders and commits, `inputRef.current` holds the actual DOM node. Then you can
call browser methods on it — `.focus()`, `.scrollIntoView()`, `.play()`.

<Callout type="note" title="React 19: ref is just a prop">
  In React 19 you can pass `ref` straight to your own function components as a normal prop — no
  more `forwardRef` wrapper. Older code uses `forwardRef` to forward a ref to an inner DOM node;
  you'll still see it, but new code doesn't need it.
</Callout>

## Use refs only for what React can't do declaratively

DOM refs are an **escape hatch** — prefer doing things with props and state. Reach for a ref
only for **non-destructive** actions React has no API for: focus, scroll, measure, media
control. Don't use a ref to add, remove, or restructure DOM nodes that React manages — you'll
be fighting the renderer, and it'll break. Let React own the DOM's structure; use refs to
*interact* with individual nodes.

## Recap

- Attach a ref to an element with **`ref={myRef}`**; after commit, `myRef.current` is the real
  **DOM node**.
- Use it to call browser methods React has no prop for: **focus, scroll, measure, media**.
- Refs are an escape hatch — don't use them to restructure React-managed DOM.
- React 19: `ref` is a normal prop (no `forwardRef` needed for new code).
