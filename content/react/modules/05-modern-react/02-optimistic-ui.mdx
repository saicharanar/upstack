When a user sends a message or likes a post, making them **wait** for the server before the UI
updates feels sluggish. **Optimistic UI** shows the expected result *immediately*, then
reconciles with the real result when it arrives. React 19's `useOptimistic` hook builds this
in — including the tricky part, the rollback.

## The idea

<Mermaid
  chart={`flowchart LR
    Act["User sends message"] --> Opt["Show it instantly (optimistic, 'sending…')"]
    Opt --> Req["Server request runs in the background"]
    Req -->|"succeeds"| Real["Replace with real, confirmed data"]
    Req -->|"fails / differs"| Revert["Auto-revert to the truth"]`}
  caption="Show the expected result immediately; React reconciles with reality when the request settles."

## useOptimistic

```jsx
import { useOptimistic } from 'react';

function Thread({ messages, sendMessage }) {
  const [optimisticMessages, addOptimistic] = useOptimistic(
    messages,
    (current, newText) => [...current, { text: newText, sending: true }],
  );

  async function action(formData) {
    const text = formData.get('message');
    addOptimistic(text);      // show it immediately, marked "sending"
    await sendMessage(text);  // the real request
  }

  return (
    <>
      {optimisticMessages.map((m, i) => (
        <p key={i}>{m.text} {m.sending && <small>(sending…)</small>}</p>
      ))}
      <form action={action}>
        <input name="message" />
        <button type="submit">Send</button>
      </form>
    </>
  );
}
```

You give `useOptimistic` the real state and a function that produces the optimistic version.
While the async Action runs, the UI shows the optimistic value; when the Action settles, React
**automatically reverts** to the real state (now updated by the server).

<Callout type="note" title="The auto-revert is the whole point">
  You don't write any rollback code. If the request fails, or the real data differs from your
  optimistic guess, React drops the optimistic value and shows the truth. It pairs naturally
  with the form Actions from the previous chapter.
</Callout>

## Where it shines

Chats, likes, reorderable lists, inline edits — anywhere the action almost always succeeds and
instant feedback matters more than waiting for confirmation. (This API needs React 19, so it's
shown here as code.)

## Recap

- **Optimistic UI** shows the expected result instantly, then reconciles with the server.
- **`useOptimistic(realState, updateFn)`** returns the optimistic value + a function to apply an
  optimistic change.
- React **auto-reverts** to the real state when the async action settles — no manual rollback.
