Here's a subtle React fact: functions defined inside a component are **created fresh on every
render**. Usually that's harmless. But if you pass such a function as a prop to a `memo`-wrapped
child, the child sees a "new" prop every render — and its memoization is defeated.
**`useCallback`** fixes this by caching the function's identity.

## The problem

```jsx
function Parent() {
  const [count, setCount] = useState(0);
  const handleClick = () => doSomething(); // a NEW function object every render
  return <MemoizedButton onClick={handleClick} />; // memo sees a "new" onClick each time → re-renders
}
```

`MemoizedButton` is wrapped in `memo`, but it re-renders on every parent render anyway, because
`handleClick` is a different function object each time.

## useCallback

`useCallback` returns the **same function** between renders, as long as its dependencies don't
change:

```jsx
import { useCallback } from 'react';

const handleClick = useCallback(() => {
  doSomething(itemId);
}, [itemId]); // same function identity until itemId changes
```

Now `MemoizedButton` receives a stable `onClick` and its `memo` works.

<Callout type="note" title="useCallback is just useMemo for functions">
  `useCallback(fn, deps)` is equivalent to `useMemo(() => fn, deps)` — one memoizes a value, the
  other memoizes a function. Same idea, same dependency-array rules; `useCallback` is the
  convenient form for functions.
</Callout>

## When to use it

The only reason to reach for `useCallback` is **stability that something depends on**:

- Passing a function to a **`memo`**-wrapped child (so its memo isn't defeated).
- Passing a function as an **Effect dependency** (so the Effect doesn't re-run every render).

<Callout type="gotcha" title="Not needed by default">
  If a function isn't passed to a memoized child or an Effect, wrapping it in `useCallback` buys
  you nothing — creating a function each render is cheap. Only reach for it to preserve identity
  that a `memo` or an Effect relies on.
</Callout>

## See it run

Click the parent button a few times. The parent re-renders each click, but the memoized `Child`
receives the **same** `onAction` function (thanks to `useCallback`), so its render count stays at
1 instead of climbing:

<LiveExample
  code={`import { useState, useCallback, memo } from 'react';

let childRenders = 0;
const Child = memo(function Child() {
  childRenders += 1;
  return <p>Child has rendered {childRenders} time(s)</p>;
});

export default function App() {
  const [count, setCount] = useState(0);
  const onAction = useCallback(() => {}, []); // stable identity across renders

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Parent clicks: {count}</button>
      <Child onAction={onAction} />
    </div>
  );
}`}
/>

Remove the `useCallback` (pass a plain `() => {}`) and the child's count would climb on every
click — a fresh function each render defeats its `memo`.

## Recap

- Functions are **re-created every render**, which can defeat a child's `memo` or re-trigger an
  Effect.
- **`useCallback(fn, deps)`** returns a **stable function identity** until `deps` change.
- It's just **`useMemo` for functions**.
- Use it only when a **memoized child** or an **Effect** depends on the function's identity.
