Right now your components render the same thing every time. To make them **reusable** — one
`Avatar` that can show *any* person — you need to pass data in. That's what **props** are: the
values a parent component hands down to a child. Props are exactly like arguments to a
function, because that's literally what they are.

## Data flows down through props

A parent passes props as attributes; the child receives them and uses them. The direction is
always the same — **down**, from parent to child:

## Passing and reading props

**Passing** looks like HTML attributes; the value can be a string in quotes or any JavaScript
in braces:

```jsx
<Avatar name="Ada Lovelace" size={64} />
```

**Reading** happens in the child. React collects all the attributes into one object called
`props`, and you pull out what you need — almost always with destructuring right in the
parameter list:

```jsx
function Avatar({ name, size }) {
  return <img alt={name} width={size} src="..." />;
}
```

That `{ name, size }` is the destructuring you learned in Module 0. (You *could* write
`function Avatar(props)` and use `props.name` — but destructuring is the norm.)

<Callout type="note" title="Props are read-only">
  A child can **read** its props but must never change them. Props are a snapshot passed down
  for this render. To make a component show something different, the **parent** passes
  different props — often driven by state, which is the next module. This one-way flow is what
  keeps React apps predictable.
</Callout>

## Default values

Give a prop a default for when the parent doesn't pass it:

```jsx
function Avatar({ name, size = 80 }) {
  // size is 80 unless the parent overrides it
}
```

## Passing JSX as `children`

When you nest JSX *inside* a component's tags, the component receives it as a special prop
called **`children`**. This is how wrapper components — cards, modals, layouts — work:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

// used as:
<Card>
  <Avatar name="Ada" size={64} />
</Card>
```

`Card` doesn't know or care what's inside it — it just wraps whatever `children` it's given.
This is one of React's most useful composition patterns.

## Recap

- **Props** pass data from a parent down to a child; they're just function arguments.
- Pass them as attributes; **read** them by destructuring in the child's parameters.
- Props are **read-only** and flow **one way** (parent → child).
- Give **defaults** with `= value`; pass nested JSX via the **`children`** prop.
