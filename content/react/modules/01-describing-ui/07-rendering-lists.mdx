Look at almost any screen: a feed of posts, a list of search results, a nav menu, a table of
rows. A huge share of every UI is **a list rendered from data**. In React you build these with
the same `.map()` you learned in Module 0 — you *map* each piece of data to a piece of markup.

## From an array of data to an array of elements

The core move: take an array, call `.map()` on it, and return a JSX element for each item.
JSX happily renders an array of elements.

<Mermaid

```jsx
const people = ['Ada', 'Alan', 'Grace'];

function PeopleList() {
  return (
    <ul>
      {people.map((name) => (
        <li key={name}>{name}</li>
      ))}
    </ul>
  );
}
```

The `{people.map(...)}` sits inside curly braces (Chapter 4 — it's a JavaScript expression),
and returns three `<li>` elements that React renders in order.

## Keys: how React tells items apart

Notice `key={name}`. Every element in a list needs a **key** — a string or number that's
**unique among its siblings** and **stable** over time. React uses keys to track which item is
which when the list changes: items added, removed, or reordered. Without keys, React warns you
in the console and can update the wrong rows (losing input focus, showing stale content).

Where does a good key come from? Your data. Use a database `id`, a unique slug, or any stable
unique field:

```jsx
{products.map((product) => (
  <li key={product.id}>{product.name}</li>
))}
```

<Callout type="gotcha" title="Don't use the array index as the key">
  It's tempting to write `key={index}`. It works *only* if the list never reorders and items
  are never inserted or removed from the middle — otherwise indexes shift, and React mismatches
  items, causing subtle bugs (wrong data in a row, lost focus). Prefer a stable id from your
  data. Reach for the index only for a truly static list.
</Callout>

## Recap

- Render lists by calling **`.map()`** on an array and returning one element per item.
- Every list item needs a **`key`** that's unique among siblings and stable over time.
- Get keys from your **data** (an `id`), not the array **index** (except for static lists).
