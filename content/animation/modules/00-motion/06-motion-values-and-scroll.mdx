Sometimes you want to drive animation from a **continuous** source — the scroll position, the
pointer, a gesture — not just from state changes. Motion's **motion values** are live numbers that
update outside React's render cycle (so no re-render per frame), and you compose them with
**`useTransform`**.

## Motion values and useTransform

A **motion value** (`useMotionValue`) holds a number that can change every frame cheaply.
**`useTransform`** derives a *new* motion value by **mapping** an input range to an output range —
this mapping is the heart of scroll and gesture effects:

```jsx
import { motion, useMotionValue, useTransform } from 'motion/react';

const x = useMotionValue(0);
// as x goes 0 → 200, opacity goes 1 → 0
const opacity = useTransform(x, [0, 200], [1, 0]);

return <motion.div drag="x" style={{ x, opacity }} />;
```

## useScroll — scroll as a value

**`useScroll`** gives you the scroll position as motion values, most usefully **`scrollYProgress`**
(0 at the top, 1 at the bottom). Feed it through `useTransform` for parallax, or use it directly
for a reading-progress bar:

```jsx
const { scrollYProgress } = useScroll();
return <motion.div style={{ scaleX: scrollYProgress, transformOrigin: 'left' }} />;
```

## Recap

- **Motion values** are live numbers that update per frame **without re-rendering**.
- **`useTransform(value, [inMin, inMax], [outMin, outMax])`** maps one value's range onto another.
- **`useScroll`** exposes scroll as motion values (`scrollYProgress`) for bars and parallax.
