Animating a list of items in *together* looks abrupt. Animating them in a **stagger** — each one
starting a fraction after the previous — is the difference between "fine" and "polished." Motion
handles the timing for you: put `staggerChildren` on the parent, give each child a variant, and
the offsets are automatic.

## staggerChildren

The parent's `visible` variant carries a `transition` with **`staggerChildren`**; each child uses
the same variant names and inherits the entrance from the parent:

```jsx
import { motion } from 'motion/react';

const list = {
  hidden: {},
  visible: { transition: { staggerChildren: 0.1 } }, // 0.1s between each child
};
const item = {
  hidden: { opacity: 0, y: 12 },
  visible: { opacity: 1, y: 0 },
};

function Skills({ skills }) {
  return (
    <motion.ul variants={list} initial="hidden" animate="visible">
      {skills.map((skill) => (
        <motion.li key={skill} variants={item}>{skill}</motion.li>
      ))}
    </motion.ul>
  );
}
```

<Callout type="tip" title="You don't compute delays">
  Without stagger you'd hand each item a `delay` and recompute when the list changes. `staggerChildren`
  derives the offsets from child order, so adding or removing items just works.
</Callout>

## Recap

- Put **`staggerChildren`** in the parent variant's `transition` to offset children automatically.
- Each child uses the **same variant names**; the parent drives them.
- Offsets come from **child order**, so the list stays correct as items change.
