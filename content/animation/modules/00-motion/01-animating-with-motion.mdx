Motion (formerly Framer Motion) gives you a set of **`motion.*` components** — `motion.div`,
`motion.button`, `motion.li` — that work exactly like their HTML counterparts but can animate.
You don't step values over time by hand; you **declare** the start and end and let Motion tween
between them.

## initial → animate

Three props do the core work:

- **`initial`** — the values the element starts at (before it appears).
- **`animate`** — the values it animates *to*.
- **`transition`** — how the motion happens (duration, easing, spring).

```jsx
import { motion } from 'motion/react';

function Card() {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}   // starts invisible, nudged down
      animate={{ opacity: 1, y: 0 }}    // fades and slides into place
      transition={{ duration: 0.5 }}
    >
      Hello, Motion
    </motion.div>
  );
}
```

<Callout type="note" title="Springs are the Motion default">
  Leave off `transition` and Motion uses a natural **spring**. Springs are defined by feel
  (`stiffness`, `damping`) rather than a fixed duration, which is why Motion animations tend to
  look alive rather than mechanical.
</Callout>

## Recap

- Use **`motion.*`** components (`motion.div`, `motion.button`, …) to make an element animatable.
- **`initial`** sets the start, **`animate`** the end, **`transition`** the how.
- Omit `transition` for a natural **spring**; set `duration`/`ease` for a timed tween.
