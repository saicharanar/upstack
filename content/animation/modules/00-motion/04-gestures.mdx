Interactive motion — a button that grows on hover, sinks on press, or an element you can drag —
usually means event handlers and state. Motion gives you **gesture props** that animate to a set
of values *while* a gesture is active, then back when it ends, with nothing to wire up.

## whileHover and whileTap

```jsx
import { motion } from 'motion/react';

function LikeButton() {
  return (
    <motion.button
      whileHover={{ scale: 1.1 }}   // grows while hovered
      whileTap={{ scale: 0.95 }}    // sinks while pressed
    >
      ♥ Like
    </motion.button>
  );
}
```

While the pointer is over the button it animates to `scale: 1.1`; while pressed, `0.95`; when the
gesture ends, it springs back — all automatic.

## drag

Add **`drag`** to make an element draggable; `dragConstraints` keeps it within bounds:

```jsx
<motion.div drag dragConstraints={{ left: -100, right: 100, top: 0, bottom: 0 }} />
```

<Callout type="tip" title="Gestures compose with variants">
  `whileHover`/`whileTap` can name variants too (`whileHover="active"`), and they propagate to
  children just like `animate`. A card can lift on hover while its label brightens — one gesture,
  coordinated children.
</Callout>

## Recap

- **`whileHover`** / **`whileTap`** animate to values while the gesture is active, then spring back.
- **`drag`** makes an element draggable; **`dragConstraints`** bounds it.
- Gesture props can reference **variants** and propagate to children.
