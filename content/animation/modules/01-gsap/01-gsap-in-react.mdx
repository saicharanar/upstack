GSAP isn't React-specific — it animates **DOM nodes directly**. So in React you give it nodes via
**refs**, and you run your GSAP code inside the **`useGSAP`** hook (from `@gsap/react`), which runs
after render, scopes any selectors to your component, and — crucially — **cleans up** every
animation when the component unmounts.

## The pattern

```jsx
import { useRef } from 'react';
import gsap from 'gsap';
import { useGSAP } from '@gsap/react';

function Box() {
  const ref = useRef(null);

  useGSAP(() => {
    gsap.to(ref.current, { rotation: 360, duration: 2 }); // animate the real node
  }); // runs after mount; auto-reverts on unmount

  return <div ref={ref} className="box" />;
}
```

<Callout type="warning" title="Always use useGSAP (not a bare useEffect)">
  You *could* call GSAP in a `useEffect`, but you'd have to track and kill every tween yourself to
  avoid leaks and double-runs. `useGSAP` does that cleanup for you and scopes selector strings to
  the component. It's the official React integration — use it.
</Callout>

## gsap.set applies values immediately

`gsap.to` animates over time. **`gsap.set`** applies values **instantly** — no tween. It's how you
establish a starting position or lay elements out:

```jsx
gsap.set(ref.current, { x: 100, opacity: 0.5 }); // applied right now, no animation
```

## Recap

- GSAP animates **DOM nodes** — reach them with **refs** in React.
- Run GSAP inside **`useGSAP(() => { ... })`**; it runs after render and **auto-cleans up**.
- **`gsap.set(node, {...})`** applies values instantly (no tween) — for starting states and layout.
