Most apps have pages like `/blog/my-post` where the last part varies. A folder named with square
brackets — **`[slug]`** — matches any value there and hands it to the page as **`params`**.

## Dynamic segments

```jsx
// app/blog/[slug]/page.tsx  →  matches /blog/anything
export default async function Post({ params }) {
  const { slug } = await params;   // e.g. "hello-world"
  return <article>Post: {slug}</article>;
}
```

## Catch-all segments

- **`[...slug]`** matches *multiple* segments (`/docs/a/b/c` → `slug: ['a', 'b', 'c']`).
- **`[[...slug]]`** is the same but also matches the base path with no segments.

## Pre-rendering with generateStaticParams

For dynamic routes you want built ahead of time (blog posts, docs), export
**`generateStaticParams`** to list the values — Next pre-renders a static page for each:

```jsx
export function generateStaticParams() {
  return [{ slug: 'hello-world' }, { slug: 'second-post' }];
}
```

<Callout type="tip" title="A breadcrumb is just the path, split up">
  Route segments and breadcrumbs are the same data seen two ways: `/blog/react/hooks` is three
  segments, and a breadcrumb trail is those segments with a cumulative link each. Turning a path
  into that trail is plain string work — which is your exercise.
</Callout>

## Recap

- **`[slug]`** captures a URL segment into **`params`**; `await params` in the page.
- **`[...slug]`** (catch-all) captures multiple segments as an array.
- **`generateStaticParams`** lists dynamic values to **pre-render** at build time.
