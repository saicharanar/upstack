You've seen how to *read* data on the server. **Server Actions** are how you *write* it — mutate
the database, then refresh the UI — without hand-building an API endpoint or a client fetch.

## A function that runs on the server

Mark a function **`'use server'`** and it runs on the server even when called from the client. Wire
it straight to a form's `action`:

```jsx
export default function NewPost() {
  async function createPost(formData) {
    'use server';                       // runs on the server
    const title = formData.get('title');
    await db.post.create({ data: { title } });
    revalidatePath('/blog');            // refresh the list
  }

  return (
    <form action={createPost}>
      <input name="title" />
      <button type="submit">Publish</button>
    </form>
  );
}
```

## Pending and errors on the client

These are the **React 19 Actions** from the React course — so `useActionState` and `useFormStatus`
work here too, giving you pending state and results with no manual wiring:

```jsx
'use client';
import { useActionState } from 'react';
const [error, submit, isPending] = useActionState(createPost, null);
```

  This is React's Actions feature (form `action`, `useActionState`, `useFormStatus`) with the
  action running on the server. If you did the React "Forms and Actions" chapter, you already know
  the client half — Next adds the server half and the `revalidate` step.
</Callout>

## Recap

- A **Server Action** is a function marked **`'use server'`** that runs on the server.
- Wire it to a **`<form action={fn}>`**; read fields from `formData`; then **revalidate**.
- The client ergonomics are React 19 **Actions** — `useActionState` / `useFormStatus`.
